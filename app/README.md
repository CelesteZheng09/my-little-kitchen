# 我的小厨房（My Little Kitchen）

一个纯前端、离线可用的个人菜谱小应用（PWA）。记录会做的菜、收藏想做的菜、看看冰箱能配什么、收藏视频教程。无后端、无账号，数据全部存在本地浏览器（IndexedDB）。

## 直接使用

需要通过 HTTP 打开（Service Worker / 模块加载要求非 file 协议）。在 `app/` 目录下任选一种本地静态服务方式，例如：

- VS Code 的 **Live Server** 插件，右键 `index.html` → Open with Live Server
- 或任意你惯用的本地静态服务器，根目录指向 `app/`

打开后即可「添加到主屏幕」，像 App 一样离线使用。

## 功能

- **厨房首页**：进入时若未选分类，默认展示全部菜品且随机排序；点分类图标筛选，右下角 ＋ 记一道菜。
- **菜品详情**：图片 + 食材 + 步骤 + 备注 + 关联教程，可编辑/删除。
- **记菜品/编辑**：菜名、封面、分类、食材、分步做法、备注。
- **想做的**：心愿清单，★ 一键移入厨房。
- **冰箱**：点选食材 →「找找能做什么」，按匹配度（命中数÷总食材数）排序，给出最佳大卡 +「还差 X 样」。
- **教程收藏**：
  - **方案 C（完整自动）**：上传本地视频/音频 → 浏览器内 Whisper 语音转文字 → 自动抽取食材/步骤（首次会下载语音模型，需联网一次，之后离线可用）。
  - **方案 B（兜底）**：粘贴分享链接/分享文案 → 先从文案里自动抽取标题、食材、步骤；配置解析代理后可抓网页内容。

## 目录结构

```
app/
├─ index.html        单页外壳（7 个视图 + 底部导航）
├─ manifest.json     PWA 配置
├─ sw.js             Service Worker（离线缓存）
├─ css/app.css       全局样式（沿用定稿 v6 白色极简）
├─ js/
│  ├─ db.js          IndexedDB 数据层 + 示例种子数据
│  ├─ match.js       冰箱食材匹配算法
│  ├─ parser.js      教程解析（本地 Whisper + 链接兜底）
│  └─ app.js         路由 + 页面渲染 + 交互
└─ icons/            本地素材（分类/食材/菜品贴纸 + 应用图标）
```

## 备注

- 顶部 ↺ 可重置为初始示例数据。
- 链接全自动解析（抖音/小红书直接抓视频）受浏览器跨域与防爬限制，纯前端无法稳定完成；如需启用，部署根目录 `worker/parse-link-worker.js`，再在 `config.js` 里填写 `PARSE_PROXY`。未配置代理时，应用仍会从复制的整段分享文案里自动抽取内容。
